# 违规情形分级与处置升级路径（研究稿）

## 一、问题意识
随着 `mils` 项目已经形成“声明—整改—撤销/终止—风险名单—后续申请强化审查—复核/解除”的授权后治理链条，一个新的制度问题进一步浮现：

在实际办理中，馆方往往不会只问“有没有违规”，而必须进一步判断：
- 该违规属于轻微偏差、一般违规、重点违规还是高风险违规；
- 目前应优先采取补正、整改，还是直接公开声明；
- 是否需要升级到撤销/终止、风险名单、暂停或限制后续申请；
- 不同工具之间是顺序升级，还是应直接并行启动。

如果制度没有把“违规情形分级”与“处置升级路径”明确写清，实践中就容易出现两类问题：
1. **处置尺度不稳**：相近情形因经办人判断不同而采取完全不同措施；
2. **升级逻辑不清**：从提醒、整改到撤销/终止、声明、名单管理之间缺少明确的跳转标准。

因此，本研究稿的核心任务，是把博物馆图片授权治理中的常见违规情形做出分级，并对应整理出一套可解释的处置升级路径。

## 二、为什么需要“分级”而不是简单列出违规类型

仅仅列出“未署名、超范围使用、擅自修改、再传播、虚假申报”等违规类型，并不足以支持治理实践。因为同一种类型在不同场景下的严重程度并不相同。例如：
- 未署名可能只是漏写来源，也可能与误导性背书、商业传播连在一起；
- 超范围使用可能只是平台范围轻微偏离，也可能是把研究用途直接转成广告营销；
- 擅自再传播可能只是个别转载，也可能是平台化分发或素材售卖。

因此，真正需要的是：
1. **违规类型识别**：发生了什么；
2. **违规严重程度判断**：影响多大、风险多高；
3. **处置路径选择**：先补正、先整改、直接声明、直接撤销/终止还是并行启动。

换言之，违规治理应从“列举行为”推进到“分级 + 升级路径”模式。

## 三、建议的违规分级框架

### （一）一级：轻微偏差 / 可即时补正事项
典型情形可包括：
- 署名格式不完全符合要求但未造成误导；
- 声明文字遗漏但尚未大范围传播；
- 成果反馈逾期但经提醒后可及时补交；
- 个别页面或单次使用环节中出现可快速修正的技术性或表达性偏差。

这一层的特点是：
- 事实明确；
- 风险外溢较小；
- 可在较短时间内纠正；
- 一般不需要动用公开声明、撤销/终止或风险名单。

对应建议处置：
1. 口头/书面提醒；
2. 要求补正、更正、补充提交成果反馈；
3. 留存处理记录；
4. 原则上不直接纳入风险名单，但可作为后续观察依据。

### （二）二级：一般违规 / 需正式整改事项
典型情形可包括：
- 未按要求署名、声明或来源标注，且已形成实际发布；
- 超出核准平台、媒介、期限或图片范围使用，但尚未形成重大传播外溢；
- 擅自进行轻度修改、裁剪、版式变形或语境偏移；
- 未按要求提交成果样本、链接或反馈，且经提醒后仍未及时履行；
- 已违反授权条件，但整体仍存在明确整改空间。

这一层的特点是：
- 已经构成正式违规；
- 需要书面化整改要求；
- 仍以纠正、补正和控制扩散为主要目标。

对应建议处置：
1. 发出整改通知；
2. 要求限期停止、删除、更正、补加声明或补办手续；
3. 视情要求补缴情形费用；
4. 对重复发生者，可纳入一般关注名单；
5. 若整改完成，通常不必立即升级到公开声明或撤销/终止。

### （三）三级：重点违规 / 需升级控制事项
典型情形可包括：
- 重复发生超范围使用、未署名、未声明、未反馈等行为；
- 擅自转载、再传播、向第三方转交或转授权；
- 将非商业、学术、教育用途实际转化为带有商业化、推广化、平台化特征的用途；
- 已收到整改要求但整改不到位、拖延整改或部分拒绝整改；
- 使用行为已造成较大范围传播，或明显增加馆方声誉、秩序或识别风险；
- 已表现出较明显的合作信用问题。

这一层的特点是：
- 不能只靠一般整改解决；
- 已进入“必须提高控制等级”的阶段；
- 处置目标从单纯纠正，转向“控制风险 + 处理授权关系 + 影响后续申请”。

对应建议处置：
1. 发整改通知并提高时限和要求；
2. 视情同步发布对外声明或严正声明；
3. 对已有授权事项评估并启动撤销/终止；
4. 纳入风险名单（一般关注或重点关注）；
5. 后续申请进入强化审查路径；
6. 对文件交付、平台范围、成果反馈、上线复核等施加更严格控制。

### （四）四级：高风险违规 / 需直接升级与并行处置事项
典型情形可包括：
- 冒名馆方、虚假合作、误导性背书；
- 未经授权进行明显商业营销、广告宣传、平台售卖或收费导览；
- 被撤销/终止后仍继续使用；
- 虚假申报、隐瞒实际使用主体、刻意规避审查；
- 大规模平台传播、素材再分发、批量导出后再上传、二次售卖；
- 已造成较大公共舆情、声誉风险或明显损害馆方管理秩序。

这一层的特点是：
- 风险高、传播快、影响大；
- 不宜仅采取“先提醒、再观察”的路径；
- 常常需要多个工具并行启动，而不是线性等待。

对应建议处置：
1. 直接发布对外声明或严正声明；
2. 直接要求停止、删除、下架；
3. 对已有授权事项直接启动撤销/终止；
4. 纳入重点关注或高度关注名单；
5. 视情暂停或限制其后续申请；
6. 必要时转入法务或更高层级处理。

## 四、违规分级时应考虑的判断因素

违规分级不宜只看行为名称，还应综合以下因素：

### 1. 行为性质
- 是技术性偏差、一般超范围，还是明显规避管理、冒名、欺骗？

### 2. 传播范围
- 是个别使用、局部页面、单平台传播，还是多平台、大范围、持续扩散？

### 3. 商业化程度
- 是否已经涉及收费、营销、广告、品牌合作、平台运营或其他营利安排？

### 4. 是否存在重复与拒不整改情形
- 是首次发生，还是多次重复；
- 是积极配合整改，还是拖延、拒不整改、规避整改？

### 5. 对授权关系与馆方秩序的影响
- 是否只是违反个别条件；
- 还是已经动摇原授权成立基础，或足以影响馆方声誉、公众识别和管理秩序？

由此可见，违规分级本质上是对“风险程度 + 传播影响 + 信用状态”的综合判断。

## 五、建议的处置升级路径

### （一）常规顺序升级路径
适用于风险由低到高逐步显现、主体仍有一定配合意愿的情形：
1. 发现偏差或违规；
2. 提醒 / 要求补正；
3. 发整改通知；
4. 整改不到位则升级至撤销/终止、风险名单；
5. 再次申请时进入强化审查；
6. 经观察和复核后，再决定是否下调或解除。

### （二）并行升级路径
适用于事项已形成公共传播影响、单一动作不足以控制风险的情形：
1. 发整改通知；
2. 同步发布对外声明/严正声明；
3. 同步评估并启动撤销/终止；
4. 同步纳入风险名单并启动后续申请限制。

这种路径强调：
- 不必机械等待“整改失败后再声明”；
- 在风险已经公开化时，公开边界与个案处置可以并行。

### （三）直接跳级路径
适用于高风险违规：
1. 不经过普通提醒或一般整改阶段；
2. 直接采取严正声明、停止下架要求、撤销/终止、名单管理等组合措施；
3. 必要时暂停后续申请并转入法务或高层审议。

这种路径的核心是：
- 对明显恶意、明显商业化、明显冒名或明显公共风险情形，不宜再以低级别工具试探性处理。

## 六、分级与工具之间的对应关系

可形成一个相对稳定的对应表：

### 1. 轻微偏差
- 主要工具：提醒、补正、记录
- 一般不启动：公开声明、撤销/终止、风险名单

### 2. 一般违规
- 主要工具：整改通知、补缴情形费用、必要记录
- 可视情启动：一般关注名单
- 一般不直接启动：严正声明、撤销/终止

### 3. 重点违规
- 主要工具：整改通知 + 风险名单 + 后续申请强化审查
- 视情启动：对外声明、撤销/终止

### 4. 高风险违规
- 主要工具：严正声明 + 停止/删除/下架要求 + 撤销/终止 + 风险名单 + 后续申请限制
- 必要时追加：法务、高层审议、专项处置

这说明：
- 工具之间不是一一对应；
- 而是随分级不同形成不同组合。

## 七、对制度草案的直接启发

### 1. “违规处理”条款可进一步体现分级治理思路
当前制度初稿已经列出多种措施，但下一步可进一步明确：
- 轻微偏差可优先补正；
- 一般违规以整改为主；
- 重点违规可同步纳入风险名单并进入后续强化审查；
- 高风险违规可直接并行启动声明、撤销/终止及申请限制。

### 2. 风险名单与强化审查不应只在“已经很严重时”出现
本研究稿提示：
- 风险名单既可用于高风险主体，也可用于重复发生一般违规的主体；
- 强化审查不是终局惩罚，而是升级治理中的中间控制工具。

### 3. “声明”与“整改”并非替代关系，而是不同维度工具
分级治理说明：
- 整改是对特定相对方提出履行要求；
- 声明是对不特定公众和平台划定边界；
- 当事项已公开化时，两者往往应同时存在。

## 八、阶段性结论
博物馆图片授权治理中的违规处置，不宜停留在“列举违规行为 + 罗列处置措施”的静态写法，而应当形成“违规分级 + 升级路径 + 工具组合”的动态治理模型。

对 `mils` 而言，本研究稿的重要推进在于：
1. 把既有的整改、声明、撤销/终止、风险名单、强化审查与复核机制，进一步放进同一套升级逻辑中；
2. 让制度草案未来不仅能回答“能采取哪些措施”，还能回答“在什么等级、什么顺序、什么条件下采取哪些措施”；
3. 使项目从模板集合和流程集合，进一步提升为一套更可解释的分级治理结构。
