# 国外博物馆开放获取与授权政策比较（第三轮）

## 一、本轮目标
在第二轮比较已形成“证据分层 + 结构化卡片”方法的基础上，本轮继续补充可稳定核验的国外官方页面，重点关注：
- 开放获取与受限授权是否并存；
- 对开放资源是否仍保留来源标注、非背书和第三方权利提示；
- 免费下载与收费高规格服务之间是否区分处理。

本轮重点补入 Rijksmuseum 与 Getty 两组更具制度代表性的案例。

## 二、本轮新增核验案例

### 1. Rijksmuseum（已核验正文）
官方页面：Information and Data Policy Rijksmuseum

本轮可确认要点：
1. Rijksmuseum 明确提出其信息与数据政策目标是尽可能广泛分享与馆藏相关的信息和数据；
2. 其基本原则是“as open as possible, and as closed as they need to be（尽可能开放，必要时才关闭）”；
3. 政策对象不只包括图像，还包括元数据、研究数据、文档和其他数字媒体；
4. 对已不受版权保护或从未受版权保护的信息与数据，可使用 Public Domain Mark（PDM）和/或 CC0；
5. 若信息与数据受版权保护但权利人为馆方，馆方通常也不行使该权利，而以 CC0 提供；仅在特定情况下才使用 CC BY 4.0；
6. 若版权属于第三方且第三方未明确放弃行使该权利，则馆方不将其作为开放数据提供；
7. 政策同时强调：尽管开放，仍认为提供 attribution/source citation 是良好实践；
8. 政策还强调可访问性、互操作性、可复用性，并明确援引 FAIR 原则。

制度启发：
- 开放政策不必只针对“图片”，还可以放在更大的“信息与数据”治理框架下；
- “尽可能开放、必要时关闭”很适合作为馆方制度写作中的总原则表达；
- 即使采用 CC0/PDM，也仍可通过“良好实践”层面鼓励来源标注，而不必一律把署名写成刚性授权前提；
- 是否开放，应由权利基础、第三方限制和安全/隐私等因素共同决定。

### 2. Getty（已核验正文）
官方页面：
- Open Content Program
- Open Content Program FAQs

本轮可确认要点：
1. Getty 自 2013 年起将大量公有领域艺术作品和档案图像纳入 Open Content Program；
2. 官方 FAQ 明确区分：
   - 已纳入 Open Content Program 且可直接免费下载的图像；
   - 因隐私、人格、合同限制、第三方版权或版权状态未研究清楚而不能免费开放的图像；
   - 不在开放计划内、需通过 Rights & Reproductions / Reproductions & Permissions 另行申请的图像；
3. Getty 对开放计划中的图像允许下载，并可提供不同尺寸；
4. 对开放计划图像，如需更大的数字文件、色彩校正或重新摄影，可另行申请并收费；
5. Getty 对 Open Content 图像在其拥有权利范围内采用 CC0；
6. Getty 明确表示：即便图像被纳入 Open Content，仍不保证不存在第三方可能主张的商标、隐私、公开权或其他权利；相关研究责任由使用者承担；
7. Getty 对开放图像的使用、修改和再利用不设限制，但明确要求不得暗示 Getty 背书或认可；
8. Getty 提供建议信用行（credit line），但不为开放内容逐案出具单独授权函；
9. 若某图像此前需许可、后续已进入 Open Content，则再次使用不需另行申请许可。

制度启发：
- 开放获取并不等于“馆方承担一切第三方权利清理责任”；
- 即使开放，也可保留“不得暗示馆方背书”的底线性限制；
- 免费开放下载与高规格、定制化服务收费可以并存；
- 对开放资源，馆方可不逐案出具授权函，而以公开政策与 FAQ 作为统一许可表达；
- 这对 `mils` 中“开放使用型 + 标准说明文本 + 额外技术服务收费”的制度组合具有直接支撑作用。

## 三、本轮比较深化结论

### 1. 开放政策并不排斥“服务收费”
Rijksmuseum 与 Getty 的做法都提示：
- 对开放资源本身可以采取较宽松的再利用政策；
- 但围绕高分辨率、重新摄影、色彩处理、专项服务等，仍可保留服务收费空间。

这对 `mils` 已形成的区分很有支撑力：
- 许可使用费 ≠ 技术/服务处理费；
- 免费开放不必等于所有文件规格和所有处理服务都无条件免费。

### 2. 开放资源仍需要权利与风险提示
本轮两个案例都强化了一个判断：
- 开放获取强调的是馆方在其可控制范围内尽量开放；
- 但并不意味着第三方权利、人格权、隐私权、商标权、敏感性风险自动消失。

因此制度上应允许：
1. 对开放资源提供统一说明文本；
2. 提示用户自行判断第三方权利风险；
3. 明确馆方不对开放资源的全部下游使用承担无限责任。

### 3. “非背书条款”是开放政策中的高频底线
Getty 的 FAQ 尤其清楚地表明：
- 即使允许任意使用、修改和再利用，也仍禁止暗示 Getty 的 endorsement（背书/认可）。

这说明：
- 非背书条款并不只是受限许可中的要求；
- 它同样适用于开放政策，是馆方维护机构立场边界和公共形象的重要底线表达。

这直接支持 `mils` 已形成的“标准署名与声明示例”继续保留并扩充非背书表达。

### 4. FAQ 是开放政策真正可执行的关键层
Getty 的 FAQ 说明，很多外部用户真正关心的问题并不是抽象许可类型，而是：
- 为什么某张图不能下载；
- 下载到什么尺寸；
- 是否可以裁剪、覆盖、印刷；
- 是否需要书面许可；
- 是否需要交样书。

这意味着：
- 后续 `mils` 项目很适合在制度草案和模板之外，再形成一份《常见问题说明（示范稿）》；
- FAQ 可成为制度的外部执行界面，减少重复沟通与解释成本。

## 四、对 `mils` 制度与模板体系的直接启发
1. 继续保持“开放使用型 / 附条件开放型 / 受限许可型 / 暂不授权型”四分法；
2. 在开放使用型中进一步细化：
   - 开放下载；
   - 高规格文件/定制服务另行收费；
   - 第三方权利风险由使用者自行注意；
   - 不得暗示馆方背书；
3. 在附录或配套文件中新增 FAQ/使用说明类文本已具备充分研究支撑；
4. 可考虑在制度说明中引入“尽可能开放、必要时关闭”的总原则表达，作为开放与风险控制平衡的上位逻辑。

## 五、本轮阶段性结论
本轮新增核验使项目获得了更强的国外对照支撑：
1. Rijksmuseum 提供了“信息与数据层面整体开放治理”的代表案例；
2. Getty 提供了“开放图像 + FAQ + 受限申请 + 技术服务收费并存”的代表案例；
3. 二者共同说明，成熟博物馆图片政策并不是简单的“开放 or 收费”，而是“权利基础判断 + 开放层级安排 + 风险提示 + 服务分层”的组合治理结构；
4. 这对 `mils` 下一步补写《常见问题说明（示范稿）》和进一步细化开放型授权条款，已经形成充分支持。
