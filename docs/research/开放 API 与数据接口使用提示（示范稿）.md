# 开放 API 与数据接口使用提示（示范稿）

> 说明：本示范稿用于博物馆官方网站、数字资源平台或开发者页面中，对开放 API、数据接口、元数据接口、下载接口或资源调用服务的使用边界进行说明。其定位不同于《批量下载与再传播提示》：后者偏页面级批量获取与再传播场景，本说明则聚焦“接口调用、程序化访问、数据对接和开发者使用”这一更具技术性的平台治理场景。正式实施时，可根据馆方接口类型、开放程度、认证机制和资源分级，对外拆分为开发者协议、接口说明页提示语、访问政策或调用申请说明。

## 一、接口开放的基本目的
本馆提供开放 API、数据接口、元数据接口或其他程序化访问方式，主要用于支持资源发现、信息检索、学术研究、公共文化传播、数字服务开发或经馆方允许的数据利用场景。

接口开放的目的，在于提升资源可发现性、可访问性和数字服务能力，并不当然意味着接口所涉及的全部资源、元数据或文件内容均可被任意抓取、复制、再传播、商用或再次授权。

## 二、接口访问不当然等于资源自由使用
### 第一条 接口开放与资源权利状态相区分
接口可访问、字段可读取、链接可调用或数据可返回，不当然改变相应图片资源、元数据或衍生文件原有的权利状态、开放程度和使用边界。

### 第二条 接口调用不当然替代授权判断
即使用户通过 API 获取到资源标识、元数据、缩略图地址、下载链接或其他信息，仍应根据具体资源页面、权利标签、开放规则和授权要求，判断其是否可直接使用、是否需申请以及是否受特别限制。

### 第三条 程序化获取不当然包含再分发权
通过接口实现抓取、缓存、同步、聚合、检索展示或数据分析，不当然意味着可将相关资源或数据再次作为自身产品、平台、接口或素材库对外提供。

## 三、建议提示的重点风险
### 第四条 权利状态混同风险
同一接口返回的数据项、资源项或字段项，可能对应不同的开放程度和使用条件。用户不得因其来自同一 API 或同一数据集，就推定均可按相同方式使用或传播。

### 第五条 大规模抓取与传播失控风险
程序化调用、批量采集、长期同步、镜像缓存和自动化分发，容易放大传播范围并削弱馆方对资源来源、语境和使用条件的控制。对大规模调用或高频抓取行为，用户应特别审慎并遵守平台限制。

### 第六条 平台替代与再许可风险
任何单位或个人不得将通过接口获取的资源、元数据或衍生文件，重新包装为自身可供公众检索、下载、调用、分发或再次授权的独立资源平台、接口平台或商业服务。

### 第七条 字段误读与上下文丢失风险
接口返回的元数据字段、标签字段、说明文本或状态值，可能只反映特定页面、特定版本或特定时间点的管理表达。用户不应脱离原页面语境、权利说明和更新状态，机械理解字段含义。

## 四、建议显示的核心提示语
### 版本 A
接口可调用，不当然意味着接口涉及的图片资源、元数据或文件内容可被任意复制、传播、商用或再次授权。

### 版本 B
同一接口返回的资源可能具有不同权利状态。请结合资源详情页、权利标签和开放规则逐项判断使用边界。

### 版本 C
程序化获取不当然包含再上传、再分发、再许可或建立替代性资源平台的权利。

### 版本 D
如拟进行大规模采集、平台级聚合、商业化使用、模型训练、跨平台传播或持续性接口转供，请先确认是否需另行申请或取得特别许可。

## 五、建议设置的确认项
如页面提供 API Key 申请、接口文档下载或开发者注册入口，可考虑要求用户确认以下事项：
1. 已知晓接口访问不等于资源自由使用；
2. 已知晓同一接口中的资源可能具有不同权利状态；
3. 已知晓程序化获取不当然包含再分发、再许可或替代平台搭建权；
4. 已知晓如涉及商业开发、平台聚合、模型训练、批量缓存、接口转供或跨平台传播，需进一步判断是否需另行申请；
5. 已知晓应尊重馆方页面规则、来源标注、限制条件、调用频率和技术管理要求。

## 六、建议配套显示的状态标签
开发者页面、接口说明页或 API 文档页可结合实际情况显示以下简明标签：
1. `接口开放不等于资源开放`
2. `同接口资源权利状态可能不同`
3. `禁止再分发/再许可`
4. `平台级聚合需另行判断`
5. `商业开发/模型训练请先确认`

## 七、适用场景提示
本说明特别适用于以下场景：
1. 开发者调用开放 API 或元数据接口；
2. 合作机构进行数据对接或批量同步；
3. 研究团队进行程序化抓取、统计分析或数字人文研究；
4. 第三方平台拟聚合展示馆方资源；
5. 企业或机构拟将相关资源嵌入产品、平台、模型或商业服务。

## 九、阶段性意义
对博物馆数字资源治理而言，开放 API 与数据接口使用提示的意义在于：在页面浏览、文件下载和批量获取之外，进一步回应数字平台时代更技术化的使用方式。它提醒外部使用者区分“接口可访问”与“资源可自由利用”之间的界限，也帮助馆方把公开展示治理、下载治理和接口治理连接起来，形成更完整的数字资源开放与授权边界体系。
