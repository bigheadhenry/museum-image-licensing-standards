# 附录模板与标准表达（研究稿）

## 一、本轮目标
在前述研究已基本覆盖授权类型、使用场景、收费减免、申请受理、审核审批、交付使用、署名声明与风险控制的基础上，本稿进一步研究制度落地所需的模板体系。

本稿重点不在形成最终格式版表单，而在回答两个问题：
- 哪些模板是制度实施中不可缺少的；
- 这些模板分别要承载哪些判断信息与控制要求。

## 二、基本判断：制度能否落地，很大程度取决于模板是否标准化
实践中，许多制度文本写得并不短，但执行效果依然不稳定，常见原因是：
- 申请表过于简单，无法支撑判断；
- 审核意见无固定结构，导致审批依据不足；
- 授权书只写“同意使用”，不写范围、限制和责任；
- 署名、声明、限制语没有标准化版本，使用人各写各的；
- 交付记录与成果反馈没有模板，事后难追溯。

因此，模板不是制度附属品，而是制度执行层。

## 三、建议纳入附录或配套文件的模板清单

### （一）申请类模板
1. **图片使用申请表**
2. **补充材料通知单**
3. **申请变更/补充说明表**

其中，图片使用申请表应至少覆盖：
- 申请主体信息；
- 实际使用主体；
- 项目名称与用途说明；
- 申请图片信息；
- 使用媒介、平台、期限、范围；
- 是否商业使用、是否收费发行；
- 是否修改、再制作、配文包装；
- 是否委托第三方执行；
- 是否申请减免及其理由；
- 承诺与签署。

### （二）审核审批类模板
1. **受理意见单**
2. **审核意见单**
3. **审批决定单**

其中，审核意见单建议固定以下栏目：
- 申请事项概述；
- 使用场景归类；
- 拟适用授权类型；
- 收费/减免判断；
- 风险点及风险等级；
- 是否需要强化审查；
- 建议审批结果；
- 建议附加条件。

审批决定单则应至少载明：
- 同意/附条件同意/暂缓决定/不同意；
- 核准用途；
- 核准范围、媒介、期限；
- 文件规格与交付前提；
- 署名、声明与限制要求；
- 违规后果提示。

### （三）授权交付类模板
1. **图片授权书/使用许可书**
2. **图片交付记录单**
3. **收费通知/减免确认单**

图片授权书应避免仅写一句“同意使用”，而应明确：
- 授权对象；
- 授权图片；
- 核准用途；
- 使用期限与范围；
- 允许与禁止的修改行为；
- 是否允许传播、转载、转授权；
- 署名和声明要求；
- 违约责任和争议处理。

图片交付记录单应明确：
- 交付日期；
- 接收主体；
- 交付方式；
- 文件规格；
- 下载或接收信息；
- 经办人；
- 备注与特别限制。

### （四）使用反馈与监督类模板
1. **成果反馈表**
2. **违规使用整改通知单**
3. **授权撤销/终止通知单**
4. **风险名单记录表**

成果反馈表可收集：
- 实际使用链接、截图、样刊、样书、片段；
- 实际发布时间、平台、版次；
- 与原审批事项是否一致；
- 是否存在修改或扩展使用情况。

### （五）标准表达类模板
1. **标准署名示例**
2. **网站页面来源标注示例**
3. **视频片尾/说明栏标注示例**
4. **非背书声明示例**
5. **转载限制提示语示例**
6. **高风险用途特别提示语示例**

## 四、重点模板的控制价值

### （一）申请表决定“能否判断”
申请表如果没有收集“实际使用主体”“传播平台”“是否修改”“是否商业合作”等信息，后续就无法真正判断风险与适用规则。

### （二）审核意见单决定“审批是否有依据”
很多审批流于形式，是因为审核意见只写“材料齐全，建议同意”。制度上应通过模板要求审核意见对用途、风险、收费与限制条件作出完整书面判断。

### （三）授权书决定“条件能否执行”
制度中的大量要求，只有进入授权书，才真正具备对外可执行性。因此授权书应成为制度规则的外化文本，而不是简单通知。

### （四）交付记录决定“事后能否追溯”
一旦发生超范围传播、外流或争议，馆方首先要能回答：交付了什么文件、交给了谁、何时交付、依据何种批准交付。没有交付记录，制度闭环难以成立。

### （五）标准表达决定“外部能否准确理解条件”
署名、声明和限制语如果没有标准示例，外部使用人往往自行简化或误写，导致制度要求在传播端失真。

## 五、可直接转化为制度条文的表达方向
关于附录模板与标准表达，可在制度中设置如下条文方向：
1. 馆方可根据授权管理需要制定并使用统一的申请、受理、审核、审批、授权、交付、反馈和整改模板；
2. 申请人应按照统一模板提交材料，并对填报内容真实性负责；
3. 经批准的事项，应通过授权书、通知书或其他书面形式明确使用条件；
4. 馆方可制定标准署名、来源说明、非背书声明和限制提示语，申请人应按要求使用；
5. 模板文本、标准表达与配套附录可作为制度实施的组成部分。

## 六、本轮阶段性结论
本轮研究形成五项可直接支撑制度起草的结论：
1. 模板体系是制度执行层，而不是附属材料；
2. 至少应形成申请、审核审批、授权交付、使用反馈与监督、标准表达五组模板；
3. 申请表、审核意见单、授权书、交付记录单是最核心的四类控制模板；
4. 标准署名、声明和限制提示语应以模板形式预设，减少外部理解偏差；
5. 正式制度草案宜将模板列入附录或配套文件体系统一管理。
