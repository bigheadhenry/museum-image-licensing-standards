# 授权申请受理、审核与审批工作流程说明（示范稿）

> 说明：本示范稿用于将图片授权申请从提交、受理、补正、审核、审批到收费确认、授权文书生成和交付前准备等环节整理为一条可执行的内部工作流程。该文本为内部工作说明，不直接替代具体申请表、审核意见单、收费确认单或授权书；办理中仍应结合《图片使用申请表》《审核意见单》《收费通知与减免确认单》《图片授权书》《图片交付记录单》及相关研究文本使用。

## 一、适用范围
本流程适用于以下情形：
1. 需要个案申请和个案判断的图片使用事项；
2. 已进入正式受理、补正、审核或审批阶段的事项；
3. 涉及收费、减免、附条件同意、暂缓决定或不同意处理的事项；
4. 需要与交付、留档和后续监督衔接的事项。

原则上，已通过公开规则明确允许直接使用的开放资源，可按公开规则办理，不必完整进入本流程；但如申请人仍需高规格文件、专项服务或超出公开规则的使用安排，仍可转入本流程。

## 二、工作目标
本流程的主要目标是：
1. 将申请受理、审核与审批从“零散环节”整理成稳定顺序；
2. 区分形式受理与实质审核，避免材料不全事项直接进入审批；
3. 把授权判断、收费/减免判断、风险控制、授权文书和交付前置条件放入同一条工作链；
4. 为后续交付、结果留档和授权后监督提供清晰的前端办理依据。

## 三、建议的流程总链条
授权申请办理可按以下链条组织：

申请提交 → 形式受理 → 补正（如需） → 实质审核 → 审批决定 → 收费/减免确认 → 授权文书生成 → 交付前核验 → 文件交付与留档

## 四、具体流程步骤

### 第一步：申请提交
申请主体提交申请时，应至少进入统一入口，并提供：
1. 主体信息；
2. 实际使用主体信息；
3. 项目名称与用途说明；
4. 拟申请图片信息；
5. 使用媒介、平台、期限、范围；
6. 是否商业使用、是否存在合作或平台传播安排；
7. 是否需要修改、再制作、批量使用或高规格文件；
8. 是否申请减免；
9. 承诺与签署。

这一阶段的重点不是直接判断是否同意，而是保证申请事项的基本要素能够进入后续受理判断。

### 第二步：形式受理
受理环节应重点判断：
1. 申请主体是否明确；
2. 使用事项是否清楚；
3. 图片对象是否可识别；
4. 使用媒介、范围、期限是否具备最低判断基础；
5. 是否足以支持后续授权类型、收费和风险判断。

如缺少核心信息，可不直接进入审核，而应进入补正程序。

### 第三步：补正（如需）
对材料不齐全、表述不清或关键事实不明确的事项，应启动补正机制：
1. 一次性告知需补充或更正的内容；
2. 明确补正期限；
3. 补正后重新进入受理判断；
4. 逾期未补正或补正后仍不足以判断的，可视为撤回或不予继续办理。

补正机制的重点，是把前端问题拦截在受理阶段，而不是把材料缺陷带入审核审批阶段。

### 第四步：实质审核
完成受理后，应进入实质审核。审核至少应围绕以下事项进行：
1. 权利基础与管理基础是否明确；
2. 使用场景归类是否准确；
3. 拟适用授权类型是否妥当；
4. 是否属于开放、附条件开放、受限许可或禁止/暂不授权；
5. 是否符合收费/减免条件；
6. 是否涉及商业合作、平台扩散、明显修改、敏感资源、复杂合作链条或其他高风险事项；
7. 是否需要设置附加条件或转入强化审查路径。

审核结果应尽量形成书面意见，为审批提供可追踪基础。

### 第五步：审查路径分流
根据审核结果，原则上可分三类路径：

#### 1. 简易路径
适用于：
- 已明确开放或低风险资源；
- 学术研究、教学、公益传播等低争议用途；
- 材料完整、用途清楚、风险较低事项。

简易路径可简化内部流转，但仍应保留基本书面记录。

#### 2. 标准路径
适用于：
- 常规出版、展览、宣传、教育等一般申请；
- 需要完成完整审核、形成明确审批意见的事项。

#### 3. 强化审查路径
适用于：
- 商业广告、品牌推广、联名合作；
- 多平台大范围传播；
- 敏感文物或限制公开资源；
- 明显改编、再制作、主视觉使用；
- 申请主体、实际使用主体、发布主体不一致；
- 可能引发法律、声誉或公共影响风险的事项。

强化审查路径可视情增加联合复核、法务支持、管理层确认或专项承诺要求。

### 第六步：审批决定
审批应在审核意见基础上作出正式决定。审批结果原则上包括：
1. 同意；
2. 附条件同意；
3. 暂缓决定；
4. 不同意。

其中“附条件同意”应尽量明确：
- 使用范围；
- 使用媒介 / 平台；
- 使用期限；
- 文件规格；
- 是否允许修改；
- 署名与声明要求；
- 是否禁止再传播、转授权或上传素材平台；
- 成果反馈要求；
- 其他风险控制条件。

“暂缓决定”适用于：
- 事实仍待补充；
- 风险仍需进一步评估；
- 需集体研究或更高层级确认的事项。

### 第七步：收费/减免确认
对经审批同意或附条件同意的事项，应根据审核和审批结果进入收费/减免确认环节，重点明确：
1. 是否不收费；
2. 是否仅收技术/服务处理费；
3. 是否收取许可使用费；
4. 是否同时收取许可使用费与技术/服务处理费；
5. 是否适用全额减免、部分减免或不予减免。

该阶段应使用《收费通知与减免确认单》等文书，将：
- 授权类型；
- 判断依据；
- 费用结构；
- 支付方式与期限；
- 减免理由与附带条件；

固定为书面结果。

### 第八步：授权文书生成
对已完成审批并完成必要收费/减免确认的事项，应生成正式授权文书，至少明确：
1. 授权对象；
2. 核准用途；
3. 使用范围与期限；
4. 文件规格；
5. 署名、声明与限制条件；
6. 禁止事项；
7. 成果反馈与责任承担要求。

这一阶段的重点是将审批结果转化为可执行、可追责的书面条件。

### 第九步：交付前核验
文件交付前，建议至少完成以下核验：
1. 审批结果是否明确；
2. 授权文书是否完成；
3. 费用是否已确认并履行；
4. 是否仍存在待补材料、待签署文本或待说明事项；
5. 交付对象、交付规格、交付方式是否与授权条件一致。

只有在上述前提基本满足后，才进入正式交付。

### 第十步：文件交付与留档
文件交付后，应至少留存：
1. 申请材料；
2. 补正记录；
3. 审核意见；
4. 审批结果；
5. 收费/减免确认；
6. 授权文书；
7. 交付记录；
8. 后续成果反馈或监督入口信息。

留档的目的不仅是归档，更是为后续使用核验、争议处理、违规处置与制度优化提供基础。

## 五、部门协同建议
实际办理中，建议形成以下基础分工：
1. 受理/承办部门：负责申请入口管理、材料初审、补正通知、流程跟踪；
2. 业务审核部门：负责用途判断、授权类型判断、风险识别、附加条件建议；
3. 收费/服务支持部门：负责费用核定、减免确认、支付核验与服务工作量说明；
4. 审批/确认层：负责同意、附条件同意、暂缓决定、不同意等正式决定；
5. 文件交付或技术支持部门：负责交付规格核验、文件准备与交付留痕。

## 六、办理中的几个关键控制点
1. **不要把申请提交直接等同于受理成功**；
2. **不要把材料齐全直接等同于可授权**，仍需完成实质审核；
3. **收费/减免判断应跟随授权判断，而不是替代授权判断**；
4. **审批通过不等于可立即交付**，仍需完成收费确认、文书生成与交付前核验；
5. **对高风险事项，应优先通过附条件同意、强化审查或暂缓决定承接风险，而不是仓促放行**；
6. **所有前端办理记录应为后续监督与违规处置留出可追溯基础。**

## 七、与其他内部文本的关系
本流程说明与其他文本的关系可理解为：
- 《申请材料与受理要求（研究稿）》：提供申请字段与受理判断基础；
- 《审核与审批流程（研究稿）》：提供审核、审批和路径分流的研究逻辑；
- 本流程说明：提供实际办理顺序；
- 《图片使用申请表》《审核意见单》《收费通知与减免确认单》《图片授权书》《图片交付记录单》：提供流程各节点的执行文书承接。

## 八、阶段性结论
授权申请办理，不应被理解为“提交表格—审批通过—直接给文件”的简化动作，而应视为一条前端治理链：
- 让申请有入口；
- 让受理有门槛；
- 让审核有判断；
- 让审批有依据；
- 让收费与减免有书面确认；
- 让授权条件有正式文本；
- 让交付前有核验；
- 让后续监督有留档基础。

只有这样，博物馆图片授权制度才能真正形成“申请前/申请中”与“授权后监督”两端衔接的完整运行结构。
