# 图片授权对外办理说明文本体系与页面分工（研究稿）

## 一、本轮目标
在项目已逐步形成 FAQ、网站规则、开放资源说明、申请说明、补正说明、限制结果说明、服务说明、页面提示语和接口提示语等一系列对外文本的基础上，本稿进一步研究：
- 这些文本之间各自解决什么问题；
- 应放在官网、数字资源平台、资源详情页、下载页、开发者页还是办理页；
- 它们之间应如何分层、分流和衔接；
- 如何避免重复写、互相冲突或页面功能混乱。

本稿的重点，不是再写单个页面文案，而是把目前已形成的大量对外文本整理成一个更清楚的“对外办理说明体系”。

## 二、基本判断：对外说明文本不应只是“很多页面”，而应是一套分层治理结构
博物馆图片授权制度如果只零散配置 FAQ、规则页和申请说明页，容易出现以下问题：
1. 用户不知道先看什么；
2. 同一问题在多个页面重复解释，但口径不一致；
3. 页面只解决“怎么申请”，却不解释“为什么要申请”“为什么会补正”“为什么不一定同意”；
4. 平台页、栏目页、详情页、下载页与正式申请页之间分工不清；
5. 对开放资源、受限资源、服务事项、接口事项的说明层次混杂。

因此，对外说明文本不应被理解为若干孤立页面，而应被理解为：
- 面向不同节点、不同用户、不同风险点配置的分层文本系统。

## 三、建议形成的七层对外说明结构
### （一）基础公开规则层
本层解决“平台上公开内容总体怎么理解”的问题，主要面向所有访问者。

建议文本包括：
1. 《版权声明与网站使用规则》；
2. 《官网平台与数字资源公开使用边界说明》。

本层主要回答：
- 官网平台是什么；
- 公开展示不当然等于自由使用；
- 一般转载、浏览、下载、来源标注、非背书和禁止事项如何理解；
- 一般网站内容与资源型栏目如何区分。

### （二）开放资源分流层
本层解决“哪些资源已开放、开放到什么程度”的问题，主要面向想直接使用资源的用户。

建议文本包括：
1. 《开放资源使用说明》；
2. 《资源详情页权利标签说明》。

本层主要回答：
- 哪些资源是开放资源；
- 开放资源不等于无条件使用；
- 即使开放，是否仍需署名、声明和遵守特别限制；
- 用户看到某个具体资源时，如何通过标签快速识别权利状态。

### （三）页面提示与动作边界层
本层解决“在栏目页、详情页、下载页、批量下载页、接口页这些高误解节点上，用户此刻该怎么理解当前动作”的问题。

建议文本包括：
1. 《数字资源栏目使用提示》；
2. 《下载页面提示语》；
3. 《批量下载与再传播提示》；
4. 《开放 API 与数据接口使用提示》。

本层主要回答：
- 浏览 ≠ 可任意使用；
- 可下载 ≠ 获得无限制许可；
- 程序化调用 ≠ 可再分发或搭建替代平台；
- 批量获取 ≠ 批量自由利用。

这一层应尽量短、准、靠近动作发生点。

### （四）正式申请入口层
本层解决“哪些事项要进入正式申请、谁可以申请、需准备什么”的问题，主要面向准备提交正式申请的用户。

建议文本包括：
1. 《图片授权申请说明页》。

本层主要回答：
- 哪些情形需申请；
- 哪些情形可能无需逐项申请；
- 谁可申请；
- 需准备哪些材料；
- 办理大致链条是什么；
- 哪些事项可能进入强化审查。

### （五）办理分流解释层
本层解决“正式申请提交后，为什么会进入补正、为什么会被暂缓、限制或不予同意”的问题，主要面向已进入办理链、需要理解程序边界的用户。

建议文本包括：
1. 《图片授权申请补充材料与补正说明》；
2. 《授权申请被暂缓、限制或不予同意情形说明》。

本层主要回答：
- 什么是补正；
- 为什么会要求补材料；
- 补正后会发生什么；
- 为什么有些事项不能被直接同意；
- 为什么要缩小范围、附加条件或强化审查；
- 哪些情形会导致不予同意或限制后续申请。

### （六）服务办理层
本层解决“能不能拿到更高规格文件、馆方是否提供额外服务、文件与服务如何收费”的问题，主要面向已进入使用或办理阶段的用户。

建议文本包括：
1. 《图片服务办理说明》；
2. 《高规格文件与专项服务说明》。

本层主要回答：
- 授权和服务不是同一件事；
- 文件规格、技术处理、批量整理、加急与纸质文书如何理解；
- 哪些事项可能单独收费；
- 为什么获批不当然等于得到最高规格文件。

### （七）外部理解与高频问答层
本层解决“用户还可能有哪些高频疑问”的问题，主要面向所有对制度整体有理解需求的用户。

建议文本包括：
1. 《常见问题说明》。

本层主要回答：
- 哪些图片能申请；
- 学术用途一定免费吗；
- 商业用途一定不能授权吗；
- 下载是否等于许可；
- 是否必须署名；
- 是否可再传播；
- 违规会有什么后果。

## 四、建议配置的页面落点
### 1. 官网全站页脚或统一规则入口
适合放置：
- 《版权声明与网站使用规则》

原因：
- 属于所有网站内容共同适用的基础规则；
- 应作为全站性通用入口存在。

### 2. 数字资源平台首页 / 平台说明页
适合放置：
- 《官网平台与数字资源公开使用边界说明》
- 《开放资源使用说明》入口链接
- 《图片授权申请说明页》入口链接

原因：
- 用户在进入资源平台前，先理解平台性质与大分流逻辑。

### 3. 数字资源栏目页 / 列表页
适合放置：
- 《数字资源栏目使用提示》

原因：
- 用户在批量浏览资源时，最需要先理解“栏目可见 ≠ 资源可自由使用”。

### 4. 单个资源详情页
适合放置：
- 《资源详情页权利标签说明》中的标签系统
- 对应的简短说明字段
- 必要时链接到《开放资源使用说明》或《图片授权申请说明页》

原因：
- 详情页是单个资源权利状态最需要被精准表达的地方。

### 5. 下载页 / 下载弹窗 / 下载确认框
适合放置：
- 《下载页面提示语》

原因：
- 下载动作是误解最容易发生的节点，必须再次区分“拿到文件”和“如何使用”。

### 6. 批量导出页 / 数据下载页
适合放置：
- 《批量下载与再传播提示》

原因：
- 用户一旦从单个下载进入批量获取，风险显著上升，应单独提示。

### 7. 开发者页 / API 文档页 / 数据接口页
适合放置：
- 《开放 API 与数据接口使用提示》
- 《开发者注册与接口申请说明》

原因：
- 接口使用者与普通页面浏览用户不同，需要单独解释接口边界与程序化风险。

### 8. 正式申请办理页
适合放置：
- 《图片授权申请说明页》
- 《图片授权申请补充材料与补正说明》链接
- 《授权申请被暂缓、限制或不予同意情形说明》链接

原因：
- 办理页不只要告诉用户“怎么填”，还要提前说明中间节点与结果边界。

### 9. 图片服务页 / 高规格文件页
适合放置：
- 《图片服务办理说明》
- 《高规格文件与专项服务说明》

原因：
- 这是“文件服务层”的独立入口，不宜与授权判断完全混写。

### 10. FAQ / 帮助中心 / 办理说明总入口
适合放置：
- 《常见问题说明》

原因：
- 作为跨层级的解释性总入口，承接高频疑问和用户搜索型浏览。

## 五、建议遵循的页面分工原则
### （一）越靠前的页面，越应解决“大边界”
例如：
- 官网平台页解决“公开展示不等于开放使用”；
- 开放资源页解决“开放到什么程度”；
- 申请页解决“哪些事项要进入办理”。

### （二）越靠近具体动作的页面，越应解决“当前动作误解”
例如：
- 下载页解决“下载 ≠ 获得无限制许可”；
- API 页解决“接口可调用 ≠ 资源可自由利用”；
- 批量导出页解决“批量获取 ≠ 批量自由分发”。

### （三）越靠近正式流程的页面，越应解释“程序节点”
例如：
- 申请页解释受理、审核、审批；
- 补正说明页解释为什么补正；
- 限制/不予同意说明页解释为什么会被限缩或驳回。

### （四）越靠近服务页，越应强调“授权与服务分离”
即：
- 获批不当然等于得到高规格文件；
- 可提供文件服务不当然等于放宽使用权。

## 六、当前体系中的几个高价值接口关系
### 1. 《图片授权申请说明页》与《图片授权申请补充材料与补正说明》
前者解决“首次提交前要知道什么”；
后者解决“提交后为什么会被要求补正”。

这是典型的：
- 入口说明 → 过程说明。

### 2. 《图片授权申请补充材料与补正说明》与《授权申请被暂缓、限制或不予同意情形说明》
前者解释“材料不足时怎么办”；
后者解释“即使材料补齐，为什么也不一定直接同意”。

这是典型的：
- 程序节点说明 → 结果边界说明。

### 3. 《开放资源使用说明》与《图片服务办理说明》
前者解决“哪些资源可直接使用”；
后者解决“即使可用，是否能拿到更高规格文件或获得服务支持”。

这是典型的：
- 使用规则 → 服务分流。

### 4. 《官网平台与数字资源公开使用边界说明》与《数字资源栏目使用提示》《资源详情页权利标签说明》《下载页面提示语》
前者是总平台说明；
后者分别下沉到栏目页、详情页、下载动作页。

这是典型的：
- 平台总规则 → 页面动作规则。

## 七、对制度原型的推进意义
本稿的意义不在于新增一个单页说明，而在于把现有大量对外文本从“若干文件”推进成“有结构的对外办理说明体系”。

它带来的推进主要有三点：
1. 让项目更清楚地区分公开规则层、开放分流层、页面提示层、申请办理层、服务办理层与 FAQ 层；
2. 让未来落地时更容易决定每份文本放在哪里，而不是都堆到一个“帮助中心”；
3. 让 `mils` 不只是写出很多制度文本，而是开始具备“平台表达架构”的原型能力。

## 八、阶段性结论
博物馆图片授权制度的对外说明，不应只是“申请说明 + FAQ”，而应形成至少七层结构：
- 基础公开规则层；
- 开放资源分流层；
- 页面提示与动作边界层；
- 正式申请入口层；
- 办理分流解释层；
- 服务办理层；
- 外部理解与高频问答层。

只有当这些文本在页面位置、解释任务和分流关系上形成清楚结构时，博物馆图片授权制度的外部表达才会真正从“零散页面集合”走向“可运行的对外办理说明体系”。
