# 交付留痕与持续监督（研究稿）

## 一、本轮目标
在已有“交付方式与使用要求”“风险控制与责任承担”“特殊情形处理”和多份执行模板的基础上，本稿进一步研究博物馆图片授权制度中容易被忽略、但对闭环治理极为关键的两个问题：
- 图片一旦交付后，馆方如何留痕；
- 图片进入实际使用后，馆方如何持续监督。

本稿目标不是建立复杂的信息系统，而是先形成一套可写入制度和模板体系的最小闭环机制。

## 二、基本判断：很多风险并不发生在审批时，而发生在交付后
在图片授权管理中，审批阶段往往最受重视，但实践中的不少问题真正出现于交付之后，例如：
- 图片交给了谁、何时交、交付何种规格，后续说不清；
- 原批准使用主体与实际接收/实际发布主体不一致；
- 已授权项目上线后增加了新平台、新语种、新页面，但馆方未被告知；
- 原本要求署名、声明、限制提示语，但上线后并未落实；
- 已要求不得转载、不得转授权，但图片最终进入了素材化扩散链条。

因此，交付留痕与持续监督应被视为授权制度的延伸执行层，而不是可有可无的补充环节。

## 三、交付留痕的制度意义

### （一）交付留痕解决的不是“存档形式”，而是责任定位
交付记录至少要回答四个问题：
1. 交付给谁；
2. 交付了什么；
3. 按什么条件交付；
4. 由谁确认接收。

如果这些问题不清，后续一旦出现超范围传播、素材外流、版本混用、责任推诿等情况，馆方就很难进行核查。

### （二）交付记录应成为授权文书与实际使用之间的连接点
申请表、审核意见、授权书分别解决“申请什么”“为何同意”“允许什么”，而交付记录解决“实际给了什么”。

没有交付记录，制度会停留在纸面批准；有交付记录，才真正形成“审批条件 → 文件交付 → 外部使用”的证据链。

## 四、交付留痕应至少记录的内容
建议《图片交付记录单》至少覆盖以下内容：
1. 对应申请编号、审批编号、授权编号；
2. 接收主体名称；
3. 实际接收人及联系方式；
4. 交付日期与交付方式；
5. 交付图片名称、编号、数量；
6. 文件规格、版本说明、是否带标识；
7. 对应授权用途、平台、期限与限制条件摘要；
8. 是否已完成收费/减免确认；
9. 是否需要成果反馈及反馈时点；
10. 接收确认与经办人签注。

这样可避免审批信息与实际交付信息脱节。

## 五、持续监督的建议结构

### （一）持续监督不等于全面监控，而是保留必要核查能力
制度中的持续监督不应理解为馆方必须逐项长期盯看，而应理解为：
- 馆方保留抽查权；
- 对重点事项设置反馈义务；
- 对高风险事项建立重点观察对象；
- 发现问题后能够及时进入整改或撤销程序。

### （二）持续监督的对象应分层
1. **一般事项**：以成果反馈、抽查和问题受理为主；
2. **中风险事项**：要求提交链接、样刊、样片或阶段性证明；
3. **高风险或特殊事项**：可要求上线前确认、上线后复核、阶段性反馈或专项台账管理。

### （三）持续监督的触发来源
持续监督可由以下线索触发：
- 申请主体按制度提交的成果反馈；
- 经办部门抽查；
- 社会公众、合作方或其他部门反馈；
- 网络检索、平台发现或馆内业务观察；
- 既往风险主体再次申请时的关联核查。

## 六、风险名单的制度作用

### （一）风险名单不是“惩罚性黑名单”的简单替代
风险名单更适合作为内部管理工具，其核心作用包括：
- 提醒经办人员注意既往不合规记录；
- 为后续申请提高审核强度提供依据；
- 区分一般疏漏、重复违规和严重违规；
- 形成案例积累和规则修正基础。

### （二）纳入风险名单的典型情形
可考虑包括：
1. 多次未按要求署名或提交成果反馈；
2. 曾发生超范围使用、擅自再传播或明显违规修改；
3. 曾虚假申报商业/非商业用途；
4. 曾拒不整改或整改不到位；
5. 曾被撤销或终止授权；
6. 其他经认定需重点关注的情形。

### （三）风险名单不应机械永久化
风险名单更适宜设置：
- 纳入原因；
- 风险等级；
- 观察期限；
- 复核结论；
- 是否解除重点关注。

这样可避免内部管理过度僵化，也更利于制度合理运行。

## 七、与整改、撤销和后续申请的衔接
交付留痕与持续监督不是独立环节，而应与以下机制联动：
- **与整改通知单衔接**：发现问题后先要求限期纠正；
- **与撤销/终止通知单衔接**：对严重或拒不整改事项启动正式处置；
- **与后续申请衔接**：风险名单主体再次申请时，应触发强化审查；
- **与特殊情形处理衔接**：高风险项目可直接纳入持续监督重点对象。

## 八、可直接转化为制度条文的表达方向
1. 馆方对已交付图片文件应形成必要交付记录，确保授权事项、交付版本、接收主体和限制条件可追溯；
2. 对已授权事项，馆方可根据风险等级要求申请主体提交成果反馈、链接、样本或其他使用证明；
3. 馆方保留对已授权事项进行抽查、核实和纠偏的权利；
4. 对多次违规、严重违规或拒不整改的主体，馆方可纳入内部风险名单管理，并在后续申请中提高审查强度；
5. 风险名单管理应记录纳入原因、风险等级、观察期限和复核结果。

## 九、本轮阶段性结论
本轮研究形成五项可直接支撑制度与模板完善的结论：
1. 交付记录是将审批条件转化为可追溯执行证据链的关键节点；
2. 持续监督的重点不是“全覆盖监控”，而是建立抽查、反馈和重点事项复核机制；
3. 风险名单应作为内部治理工具，而非简单惩罚性名单；
4. 交付留痕、整改、撤销/终止和后续申请强化审查应形成联动闭环；
5. 本轮成果可直接支持制度初稿增加“交付留痕与持续监督”条款，并补齐相关示范模板。
