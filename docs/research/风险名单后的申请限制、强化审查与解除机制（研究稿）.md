# 风险名单后的申请限制、强化审查与解除机制（研究稿）

## 一、问题意识
在 `mils` 当前制度原型中，风险名单记录已经被明确为一类持续监督工具，用于记录重复违规、严重违规、拒不整改、虚假申报、擅自商业化扩散或被撤销/终止授权的主体。但如果制度只写到“纳入风险名单”为止，仍然存在一个关键缺口：

被纳入风险名单之后，该主体后续再来申请图片授权时，馆方应如何处理？

如果没有进一步规则，就容易出现两类问题：
1. **名单记录失去实际约束力**：主体虽然被纳入风险名单，但后续申请与普通申请几乎没有差别，导致内部留痕与风险管理流于形式；
2. **风险控制缺乏边界**：馆方如果完全凭临时判断处理后续申请，又容易出现尺度不一、标准不清、难以解释甚至争议升级的问题。

因此，本研究稿的核心任务，是把“纳入风险名单之后怎么办”写清楚，形成从风险记录到后续申请管理、再到复核调整与解除的闭环机制。

## 二、风险名单的制度定位应当从“记录工具”扩展为“后续申请管理入口”

风险名单不应仅被理解为内部备忘录，而应视为后续申请处理中的一个前置识别节点。其制度功能至少包括：
1. **识别功能**：提醒馆方该主体存在既往风险记录；
2. **分流功能**：决定后续申请是否进入强化审查、专项审查或暂缓受理路径；
3. **控制功能**：对高风险主体提高说明、承诺、审核与交付控制要求；
4. **恢复功能**：在整改到位、观察期届满且复核通过后，允许调整或解除关注状态；
5. **解释功能**：使馆方能够对“为什么这次申请需要更严格审查”给出制度化说明。

换言之，风险名单不是终点，而是后续申请治理的起点。

## 三、风险等级与后续申请处理路径

当前 `风险名单记录表（示范稿）` 已将风险等级分为：
- 一般关注
- 重点关注
- 高度关注

这一分级应当对应不同的后续申请处理路径。

### （一）一般关注：进入“提示性强化审查”
适用情形可包括：
- 首次出现较轻违规；
- 已完成整改但仍需观察；
- 未按时反馈成果但后续已补交；
- 其他虽存在问题但整体可控的情形。

后续申请建议处理方式：
1. 在受理环节自动提示承办人注意其既往记录；
2. 要求申请主体在新申请中补充说明既往问题及后续改正情况；
3. 对使用场景、平台、传播范围和是否涉及第三方合作进行更细审查；
4. 必要时要求补充承诺文本；
5. 原则上仍可正常进入审批，但不适用最简化路径。

其核心不是“拒绝”，而是“带着既往风险意识进行审查”。

### （二）重点关注：进入“强化审查 + 附加条件控制”
适用情形可包括：
- 多次违规；
- 曾发生超范围使用、擅自转载、擅自修改等较明显问题；
- 已收到整改通知；
- 曾造成一定传播影响；
- 主体合规性存在持续疑问。

后续申请建议处理方式：
1. 不适用简化受理或快速审批路径；
2. 应补充提交更完整的项目说明、传播方案、合作链条说明或平台信息；
3. 视情要求提交专项承诺书；
4. 进入部门负责人复核、联合审核或专项审核路径；
5. 可缩小核准范围、压缩期限、减少平台、限制修改、提高署名与声明要求；
6. 可要求更严格的成果反馈、上线后复核或分阶段交付；
7. 必要时提高服务环节控制，如不直接提供高规格文件或限制批量交付。

其核心不是当然拒绝，而是“在允许重新申请的前提下，提高授权门槛与过程控制”。

### （三）高度关注：进入“暂缓、限制或从严决定路径”
适用情形可包括：
- 拒不整改；
- 冒名活动；
- 虚假申报；
- 被撤销/终止授权后仍继续使用；
- 造成重大传播失控、较大声誉风险或明显恶意规避行为；
- 其他显示合作信用严重不足的情形。

后续申请建议处理方式：
1. 可暂停受理一定期限内的新申请；
2. 即使受理，也应进入最高等级强化审查，不适用简化路径；
3. 原则上须由更高层级审批或专项集体审议；
4. 对高风险使用场景（商业宣传、广告营销、平台传播、批量使用、再制作等）可从严控制或原则上不予同意；
5. 可要求先完成历史问题处置、删除下架、费用补缴情形、书面致歉、专项承诺等前置事项，再决定是否继续办理；
6. 对风险未明显降低前，可维持“暂缓受理”或“不予同意”的处理结果。

其核心是：风险名单中的“高度关注”不只是注意观察，而是应真实影响后续申请处理结果。

## 四、后续申请中的强化审查要素

当风险名单主体再次申请时，强化审查不应停留在“多看一眼”，而应形成相对稳定的审查要素。至少包括：

1. **既往问题复盘**
   - 曾发生什么问题；
   - 是否已经整改；
   - 是否再次出现类似行为；

2. **主体信用判断**
   - 申请陈述是否一致；
   - 是否继续隐瞒第三方、实际使用方或平台安排；
   - 是否具备稳定履行条件的能力；

3. **用途与传播风险再评估**
   - 是否仍申请高传播、高商业化或高扩散场景；
   - 是否涉及跨平台分发、合作推广、批量使用或可再传播链条；

4. **控制条件是否足以覆盖风险**
   - 单靠署名/声明是否足够；
   - 是否需要缩小授权范围；
   - 是否需要提高交付控制；
   - 是否需要更强成果反馈与上线后复核；

5. **是否需要继续限制甚至不予同意**
   - 如历史问题未解决、整改不到位、解释不充分或风险明显重复，应允许馆方作出从严决定。

## 五、解除与调整机制为什么重要

如果制度只写“纳入风险名单后从严处理”，但没有解除与调整机制，也会产生问题：
1. 主体即使已经整改到位，也难以恢复正常申请状态；
2. 风险名单容易变成无限期标签，不利于制度公正性；
3. 馆方内部也缺少明确标准，不知道何时可以下调风险等级。

因此，风险名单制度应同时包含：
- **纳入机制**
- **后续申请限制机制**
- **复核机制**
- **调整/解除机制**

这四部分缺一不可。

## 六、建议的复核与解除条件

### （一）可启动复核的基本前提
被纳入风险名单的主体，如符合以下条件之一，可申请或进入馆方主动复核：
1. 已按要求完成整改并提交证明材料；
2. 已停止违规行为并删除、下架、补正相关内容；
3. 已补缴情形费用或履行其他责任；
4. 在观察期内未再发生新的违规情形；
5. 能合理说明历史问题成因并提出改进措施；
6. 其他足以表明风险已下降的情形。

### （二）复核时应重点判断的事项
1. 历史问题是否真正整改到位；
2. 相关传播内容是否已删除、更正或失效；
3. 主体是否表现出真实合规意愿；
4. 后续管理措施是否已经建立；
5. 风险是否实质下降，而不仅是形式说明。

### （三）复核结果建议分为三类
1. **维持原等级**：整改不到位、风险仍高；
2. **下调等级**：风险有所降低，但仍需观察；
3. **解除重点关注/移出名单**：整改完成、观察期满、后续行为稳定，已无继续维持高关注的必要。

## 七、制度条文层面的直接启发

### 1. 风险名单应与“暂停或限制后续申请”条款形成明确连接
当前制度初稿已写到馆方可“暂停或限制其后续申请”，但下一步应进一步明确：
- 该措施可与风险名单联动；
- 不同风险等级对应不同后续申请处理路径；
- 不是一律拒绝，而是分级控制。

### 2. 应将“复核与解除机制”写入制度或配套模板
如果后续制度文本中只写“纳入名单”，不写“如何复核、何时解除”，将导致制度不完整。因此，建议：
- 在制度正文中增加风险名单后的管理与复核条款；
- 或在 `风险名单记录表` / 其他配套模板中增加观察期、复核意见、等级调整与解除记录。

### 3. 风险名单后的控制不应仅体现在审批上，也应体现在交付与监督上
被列入重点关注或高度关注的主体，即便重新获批，也不应简单沿用普通交付路径，而应考虑：
- 限制文件规格；
- 限制批量交付；
- 增加成果反馈要求；
- 要求上线后复核；
- 缩短授权期限或缩小平台范围。

这说明“后续申请限制”不是单一审批动作，而是贯穿受理、审核、审批、交付与后监督的整链控制。

## 八、阶段性结论
风险名单制度只有在能够真实影响后续申请处理，并且又保留复核、调整、解除路径时，才是完整制度，而不是单纯内部记录。

对 `mils` 而言，当前最重要的推进是：
1. 把风险名单从“记录表”推进为“后续申请分流机制”；
2. 把分级控制与强化审查写入制度草案；
3. 把复核与解除机制补齐，以保持制度的可执行性与可解释性。

这样，项目就能从“发现问题后怎么处置”，进一步推进到“处置后如何持续管理以及何时恢复正常管理”的完整治理闭环。
