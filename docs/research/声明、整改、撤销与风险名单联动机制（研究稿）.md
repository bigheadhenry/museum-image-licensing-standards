# 声明、整改、撤销与风险名单联动机制（研究稿）

## 一、问题意识
随着 `mils` 项目逐步形成“版权声明/网站规则—栏目与详情提示—下载与接口提示—申请与服务办理说明—授权/整改/撤销/风险名单”等多层制度原型，一个新的执行性问题逐渐明确：

当博物馆发现图片资源、导览资源、展览信息或相关公共服务资源被未经授权商业利用、冒名活动、误导性传播或其他方式越界使用时，馆方究竟应如何在不同文本和不同处置动作之间形成衔接，而不是零散回应？

尤其是在国内实践中，很多治理边界并不只体现为一页静态“图片使用政策”，而是通过：
- 对外声明/严正声明；
- 个案整改通知；
- 授权撤销或终止通知；
- 内部风险名单记录；
- 后续强化审查与持续监督；
共同形成一个分层处置体系。

因此，本研究稿的核心目的，是把这几类文本与动作之间的关系明确化，形成可执行的联动逻辑。

## 二、四类工具的功能定位

### 1. 对外声明 / 严正声明
**主要功能：**
- 面向社会公众、平台、媒体、合作方和潜在使用者公开划定边界；
- 澄清事实；
- 说明馆方正式口径；
- 提示未经授权商业利用、冒名活动或误导传播的性质；
- 维护馆方权益与社会声誉；
- 向外部释放“馆方已注意到并已正式回应”的信号。

**主要适用条件：**
- 事项已经进入公共传播空间；
- 已对公众认知、平台秩序、交易安排或馆方声誉产生影响；
- 需要面向不特定对象统一说明边界。

### 2. 整改通知
**主要功能：**
- 面向特定相对方提出限期改正要求；
- 促使其停止违规、删除内容、补正署名、补加声明、补办授权或提交说明；
- 将“公开边界说明”转化为“个案履行要求”。

**主要适用条件：**
- 对象明确；
- 问题可定位；
- 尚存在改正空间；
- 馆方希望优先促成停止、纠正或补正。

### 3. 授权撤销或终止通知
**主要功能：**
- 对已经获得授权但后续严重违规、条件不再满足或项目发生重大变化的事项，作出正式撤销/终止决定；
- 明确自何时起不得继续使用；
- 要求删除、下架、停止传播并承担相应后果。

**主要适用条件：**
- 违规严重；
- 整改不到位；
- 使用已经明显超出原批准边界；
- 或者原有授权前提已经发生实质变化。

### 4. 风险名单记录
**主要功能：**
- 面向馆内的持续监督工具；
- 记录重复违规、严重违规、拒不整改、冒名活动、虚假申报或被撤销/终止授权的主体；
- 支持后续强化审查、限制受理、提高审核门槛或持续观察。

**主要适用条件：**
- 需要长期跟踪而不仅是单次处置；
- 行为反映出较高复发风险、规避倾向或合作信用问题；
- 已具备纳入内部风险管理的事实基础。

## 三、四类工具之间的关系

### （一）它们并不是彼此替代，而是不同层级的处置工具
可将其理解为以下四个层次：

1. **公开边界层**：对外声明 / 严正声明
   - 解决“社会上怎么说、公众怎么理解、平台如何识别”的问题；
2. **个案纠正层**：整改通知
   - 解决“具体对象需要做什么改正”的问题；
3. **授权关系处理层**：撤销 / 终止通知
   - 解决“原有授权是否继续有效、能否继续使用”的问题；
4. **持续监督层**：风险名单记录
   - 解决“该主体以后还应如何被管理”的问题。

### （二）它们可以串联，也可以并行
在实践中，这四类工具既可能按顺序出现，也可能并行使用。

#### 情形 1：典型顺序链
适用于“问题先被发现 → 先要求改 → 改不好再升级”的情形：
1. 发现问题；
2. 向特定主体发出整改通知；
3. 逾期不改或再次违规，撤销/终止授权；
4. 纳入风险名单；
5. 如事件已造成公共传播影响，可同步或随后发布对外声明。

#### 情形 2：公开传播已形成时的并行链
适用于“事情已经公开扩散、仅靠私下整改不足以澄清”的情形：
1. 馆方先发布声明/严正声明，统一划定边界；
2. 同时向具体主体发出整改通知；
3. 对已取得授权但越界使用者，启动撤销/终止程序；
4. 对拒不整改或高风险主体纳入风险名单。

#### 情形 3：高风险情形的直接升级链
适用于“冒名活动、虚假合作、平台售卖、广泛收费导览、严重声誉损害”等明显高风险情形：
1. 直接发布严正声明；
2. 直接发出停止、删除、下架要求；
3. 必要时不经过普通补正阶段，直接作出撤销/终止或限制后续申请决定；
4. 纳入内部重点关注或高度关注名单。

## 四、联动触发条件建议

### 1. 触发“对外声明/严正声明”的典型条件
- 已在网络平台形成较大范围传播；
- 已对公众造成明显误导；
- 已出现“本馆授权 / 本馆合作 / 官方讲解 / 官方指定”等误导性表述；
- 已对馆方声誉、秩序或公众识别造成影响；
- 需要统一向不特定对象说明边界。

### 2. 触发“整改通知”的典型条件
- 已找到具体主体；
- 能明确指出其违规行为；
- 该行为仍有纠正空间；
- 馆方希望通过限期整改先行控制风险。

### 3. 触发“撤销/终止通知”的典型条件
- 已获授权但严重越界使用；
- 未按要求整改或整改不到位；
- 项目、主体、传播平台、用途发生重大变化；
- 已失去原授权条件成立基础；
- 行为继续存在将产生更大风险。

### 4. 触发“风险名单记录”的典型条件
- 多次违规；
- 拒不整改；
- 虚假申报；
- 擅自商业化扩散；
- 擅自再传播/转授权；
- 被撤销/终止授权；
- 存在较高复发风险或合作信用风险。

## 五、建议的馆内处置流程

### （一）基本研判步骤
1. **先确认事实**：核实页面、传播内容、交易信息、使用方式、时间范围和相关主体；
2. **判断影响范围**：是个案违规，还是已经进入公共传播空间；
3. **判断主体状态**：是未申请的外部主体，还是已有申请/授权关系的主体；
4. **判断风险等级**：一般、重点、高度；
5. **决定工具组合**：声明、整改、撤销/终止、风险名单是否需要同时启动。

### （二）推荐的动作组合

#### 1. 一般违规但尚可纠正
- 先发整改通知；
- 留存整改反馈；
- 如整改完成，一般不必公开声明；
- 是否纳入风险名单，视是否重复违规而定。

#### 2. 已产生社会传播影响
- 发对外声明/严正声明；
- 同时发整改通知；
- 如涉及已有授权事项，评估是否撤销/终止；
- 视情纳入风险名单。

#### 3. 已构成严重越界或冒名活动
- 直接发严正声明；
- 直接要求停止、删除、下架；
- 直接启动撤销/终止或限制后续申请；
- 纳入重点关注或高度关注风险名单；
- 必要时转入法务或更高层级处置。

## 六、对制度原型的启发

### 1. 馆方外部表达层应当包含“公开处置文本”
`mils` 当前的制度原型已经有：
- 一般规则文本；
- 页面提示文本；
- 办理入口文本；
- 个案文书模板；

本研究稿进一步说明：
- 还应把“公开处置文本”明确作为一层，即对外声明/严正声明；
- 它与内部文书并不是重复关系，而是分别处理“社会传播边界”和“个案纠正执行”。

### 2. 风险控制应从“授权前审查”扩展到“授权后治理链条”
此前 `mils` 已经逐步把项目从“申请、审核、收费、授权、交付、反馈”扩展到“整改、撤销/终止、持续监督”。本研究稿进一步明确：
- 这些动作之间不是平铺并列，而是可以形成升级链和联动链；
- 这意味着制度草案后续可以更明确地区分：
  - 公开澄清；
  - 个案整改；
  - 授权关系处理；
  - 后续持续监管。

### 3. 国内馆治理逻辑更适合“分布式规则 + 联动处置”模型
对比国外较常见的集中式版权页/授权页表达，国内实践更常出现：
- 分散在官网、栏目、声明、公告、联系页中的边界表达；
- 出现问题后再通过声明、整改、撤销与内部名单形成联动。

这说明 `mils` 当前形成的制度原型，越来越接近国内可执行模式，而不是停留在抽象的授权规则总结。

## 七、阶段性结论
“声明—整改—撤销/终止—风险名单”并不是四份互不相关的文书，而是博物馆图片授权治理中从公开边界表达、到个案纠正、到授权关系处理、再到持续监督的一个处置链条。

对 `mils` 而言，下一步最自然的制度推进，不只是继续增加单份模板，而是把这条联动链明确写入制度草案，使制度文本本身能够解释：
- 什么情况下需要公开声明；
- 什么情况下先整改；
- 什么情况下升级到撤销/终止；
- 什么情况下纳入持续风险管理。

这将使项目从“模板集合”进一步提升为“可解释的联动治理机制”。
